Proyecto Urban.Routes - Testing de Automóvil Compartido y Taxi Aéreo

Durante el proyecto Urban.Routes, realicé pruebas exhaustivas de las funcionalidades de "Automóvil Compartido" y "Taxi Aéreo", garantizando estabilidad y una experiencia de usuario fluida.

  • Automóvil Compartido: Validación del formulario de reserva en Google Chrome (800x600) y Firefox (1920x1080). Se diseñaron y ejecutaron casos de prueba para el botón "Reservar", ventanas emergentes y procesos de alquiler, cubriendo pruebas positivas y negativas.
  • Taxi Aéreo: Uso de Charles Proxy para simular el backend, permitiendo pruebas anticipadas del front-end. Validación del ícono, cálculo de costos y presentación de resultados para asegurar consistencia visual y funcional.

El proyecto concluyó con un reporte detallado de errores y mejoras, asegurando que la plataforma cumpla con altos estándares de calidad, fortaleciendo la confianza del usuario y recomendando su despliegue.

Proyecto Urban.Grocers - Pruebas de API

Llevé a cabo pruebas exhaustivas del back-end de Urban.Grocers, asegurando la correcta integración de nuevas funcionalidades relacionadas con kits de productos, servicios de entrega y el manejo del carrito de compras.

  • Manejo de Kits: Probé la capacidad de agregar productos a un kit mediante el endpoint POST /api/v1/kits/{id}/products, validando respuestas exitosas y errores 400 Bad Request.
  • Servicios de Entrega: Validé la disponibilidad y el costo del servicio Order and Go usando el endpoint POST /order-and-go/v1/delivery, asegurando que el cálculo de precios fuera preciso según los requisitos.
  • Gestión del Carrito:
    • Obtención de listas de productos con GET /api/v1/orders/:id, verificando la autenticación correcta para evitar errores 401 y 404.
    • Pruebas de adición de productos al carrito con PUT /api/v1/orders/:id, manejando escenarios de disponibilidad y errores 409 Conflict cuando no se encuentran productos en almacenes.
    • Eliminación del carrito usando DELETE /api/v1/orders/:id, asegurando respuestas correctas y manejo de errores.

Utilicé Postman para ejecutar todas las pruebas API y reporté errores encontrados en Jira, lo que permitió mejorar la calidad del producto. Al finalizar, presenté un informe detallado sobre el estado de las pruebas para garantizar un despliegue exitoso.

Proyecto Urban.Lunch - Pruebas de Aplicación Móvil

En este proyecto, realicé pruebas exhaustivas de la aplicación móvil Urban.Lunch para asegurar su funcionamiento en dispositivos Android. El enfoque se centró en la creación de listas de comprobación detalladas para validar las funcionalidades clave, utilizando mapas mentales y diagramas de flujo como soporte.

Las pruebas se llevaron a cabo tanto en un emulador de Android Studio como en dispositivos físicos, con la última versión de la aplicación. Los resultados de cada prueba se marcaron como APROBADO o NO APROBADO. Los errores detectados fueron documentados y reportados en Jira, incluyendo enlaces directos a los informes para facilitar su revisión.

  • Creación de la lista de comprobación: Diseñé listas de pruebas exhaustivas para cubrir las funcionalidades principales de la aplicación, asegurando una verificación minuciosa.
  • Ejecución de pruebas: Las pruebas se realizaron en diferentes entornos Android, verificando la estabilidad y la correcta interacción del usuario con la interfaz.
  • Documentación de errores: Todos los errores encontrados fueron reportados detalladamente en Jira, proporcionando evidencias claras para su corrección.

Al concluir el proyecto, redacté un informe de pruebas detallado para el equipo, destacando el estado de las funcionalidades probadas, los errores encontrados y recomendaciones para su corrección. Esto garantiza que la aplicación cumpla con los estándares de calidad necesarios para su lanzamiento exitoso.

Proyecto Urban Routes - Automatización de Pruebas

Este proyecto consiste en la creación de pruebas automatizadas para la plataforma Urban Routes, utilizando Git, GitHub y Python. Se implementaron varias pruebas para validar el proceso de solicitud de un taxi y garantizar una experiencia de usuario fluida y confiable.

Configuración del Proyecto

El proyecto fue gestionado con Git y GitHub, asegurando control de versiones eficiente. Tras clonar el repositorio, el proyecto se configuró localmente utilizando PyCharm para un entorno de desarrollo optimizado.

Implementación de Pruebas Automatizadas

Las pruebas se implementaron en main.py, utilizando la estructura de clases para mantener el código modular y limpio. La clase UrbanRoutesPage define los localizadores y métodos necesarios para interactuar con la aplicación, mientras que la clase TestUrbanRoutes contiene las pruebas automatizadas que cubren funcionalidades de la aplicación.

Creación del Archivo README

Se añadió un archivo README.md al repositorio, incluyendo una descripción detallada del proyecto, las tecnologías utilizadas y las instrucciones para ejecutar las pruebas. Esto facilita la comprensión y ejecución del proyecto por parte de otros desarrolladores.
